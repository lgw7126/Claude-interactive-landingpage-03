# 월하재 月下齋 — 인터랙티브 한옥스테이 랜딩페이지

**▶ [지금 바로 실행해보기](https://claude.ai/code/artifact/2302ac3e-797c-4072-9d6e-dd9a5a810de5)**

처마 끝에 매달린 한글 시구(詩句)의 발을 커서로 흔들고, 스크롤로 걷어 올리는 한옥스테이 랜딩페이지입니다. 순수 `Canvas 2D` + 바닐라 JS로 만들어졌고, 외부 라이브러리가 전혀 없습니다.

---

## 로컬에서 실행하기

이 저장소를 클론한 뒤 `index.html`을 브라우저로 바로 열면 됩니다. 별도의 빌드나 서버가 필요 없습니다.

```bash
git clone https://github.com/lgw7126/Claude-interactive-landingpage-03.git
cd Claude-interactive-landingpage-03
open index.html   # 또는 더블클릭
```

GitHub Pages를 켜면 (Settings → Pages → Branch: main) 별도 배포 없이 공개 URL로도 볼 수 있습니다.

## 제작 과정

### 1. 레퍼런스 영상 분석
카자흐 유르트·중국 자금성·일본식 지붕 아래로 각국 문자가 커튼처럼 드리워지고, 마우스 인터랙션에 파도치듯 반응하는 여행 랜딩페이지 영상(`interactive-landingpage.mp4`)에서 출발했습니다. 이 세션은 네트워크가 제한된 원격 환경이라 영상 코덱을 직접 디코딩할 수 없었는데, Chromium의 `ImageDecoder` API로 GIF 프레임을 추출해 우회했습니다.

### 2. 컨셉 재해석
동일한 "글자 커튼 + 커서 반응" 문법을 그대로 베끼는 대신, 완전히 다른 소재로 재해석했습니다 — **한옥 처마와 한글 발(簾)**. 대나무 발 대신 윤동주 「서시」와 김소월 「산유화」를 세로로 엮어, 발이 곧 시가 되도록 했습니다.

### 3. 물리 엔진 반복 개발
글자 하나하나가 파티클인 커튼의 움직임을 세 번에 걸쳐 다시 만들었습니다.

| 단계 | 방식 | 문제 |
|---|---|---|
| v1 | 개별 스프링 (홈 포지션 + 반발력 + 감쇠) | 퉁퉁 튕기는 인공적인 움직임 |
| v2 | Verlet 로프 체인 (거리 제약 조건) | 여전히 미세하게 진동, 손으로 짠 느낌 부족 |
| v3 | **속도 없는 드레이프(늘어짐) 모델** | 구조적으로 튕김이 불가능 — 각 글자가 "위 글자 아래 매달린 자리"를 향해서만 천천히 이징 |

최종 버전은 레퍼런스 GIF를 프레임 단위로 다시 분석해 나온 결과입니다: 커서는 글자를 밀어내는 게 아니라 **그립(붙잡음)** 개념으로 끌고 다니며, 놓았을 때 위에서부터 아래로 몇 초에 걸쳐 순차적으로 정착합니다. 상수 튜닝은 Node.js로 물리 시뮬레이션을 독립 실행해 수렴 속도를 미리 계산한 뒤 반영했습니다.

### 4. 지붕 아트워크 교체
처음엔 SVG로 기와·수막새·용마루를 절차적으로 그렸지만 도식적인 한계가 있었습니다. 사용자가 이미지 생성 AI로 만든 단청 처마 렌더를 받아, 배경을 코드로 제거(경계 flood-fill)하고 WebP로 압축해 페이지에 내장했습니다. 처마 밑선은 이미지의 알파 채널을 열 단위로 스캔해 자동 추출하므로, 다른 지붕 이미지로 교체해도 발이 그 곡선을 그대로 따라갑니다.

### 5. 디테일 튜닝
글자 크기와 간격, 커서에 끌려 올라갔다가 떨어지는 낙하 속도, 가닥마다 간격·먹 농담을 미세하게 다르게 주는 것까지 — 사용자 피드백을 받아가며 여러 차례 다시 다듬었습니다.

## 기술 스택

- **HTML/CSS**: 시맨틱 마크업, 라이트/다크 테마 토큰 분리, 반응형 레이아웃
- **Canvas 2D**: 글자 커튼 렌더링 및 물리 시뮬레이션 (외부 라이브러리 없음)
- **폰트**: Gowun Batang(세리프 헤드라인), Gowun Dodum(본문), IBM Plex Mono(라벨)
- **접근성**: `prefers-reduced-motion` 대응, 키보드 포커스 표시, ARIA 라벨
